# Project Dao.Science 深度审计报告

> 审计时间：2026-06-13  
> 审计范围：全仓库 24 个 Markdown 内容模块、8 篇 LaTeX 预印本、MkDocs 工程配置、开源治理文件  
> 审计视角：内容架构 / 科学严谨性 / 技术工程 / 传播可及性 / 独家战略判断

---

## 一、总体诊断

Project Dao.Science 已经越过了“灵光一现”的阶段，进入了一个更危险也更关键的区间：**它看起来像一门科学，但尚未建成让外界可以独立检验它的基础设施。**

项目目前最突出的矛盾是：

> **形式化丰富，但操作化不足；**
> **概念地图精美，但验证链路断裂；**
> **宣言充满力量，但工程最后一公里尚未打通。**

这不是贬低。恰恰相反，项目已经拥有了高水准的跨学科整合野心、清晰的叙事骨架和自反性的批判意识（`objections_and_replies.md` 是亮点）。但它现在同时面临两个风险：
1. **学术风险**：核心公式若停留在“隐喻包装”，会被严肃同行评议快速驳回；
2. **传播风险**：工程与可及性缺陷会让潜在读者、贡献者、合作者难以进入。

最深的改进点因此不是“再加几个模块”，而是：**为每一个核心命题建立一个可回答的接口——形式、仿真、实验、工程四位一体。**

## 二、四大深层问题（附证据与建议）

### 问题 1：核心公式的“数学神圣化”——隐喻与模型的边界模糊

**问题本质**：项目把 `道 ≡ -∇_π G(π)` 作为第一公理，并宣称“所有核心命题都有数学形式化”。但这个等式不是对“道”的操作化定义，而是主动推理中策略选择概念与道家概念之间的一个**形式类比**。更严重的是，数学处理本身存在问题：

- `π` 在标准主动推理中通常是**离散策略索引**，对离散下标写 `∇_π G(π)` 没有标准数学意义；
- 同一段落里又使用 softmax 离散策略选择 `P(π)=σ(-γG(π))`，与“对 π 求梯度”的连续图景冲突；
- 式 (13) `P(π_顺道)=σ(-γ·min_π G(π))` 把全局标量喂入 softmax，导致所有策略概率相同，无法表达“选择最小 EFE 策略”。

**影响**：这会让任何受过形式训练的认知科学家/数学家立刻质疑：项目是在做理论，还是在做修辞？`objections_and_replies.md` 自己也承认公式是“操作化指针”而非定义，但 `POSITIONING.md` 和 `GLOSSARY.md` 仍用 `≡` 和“所有核心命题都有数学形式化”来宣传，形成张力。

**建议**：
1. 在全项目区分 **F（纯形式化/隐喻）/ S（有仿真）/ B（有行为预测）/ N（有神经证据）/ M（元伦理/规范性主张）** 五级证据状态；
2. 把 `Dao ≡ -∇_π G(π)` 重写为两种可辩护的形式之一：
   - 连续参数化策略：`Dao(θ) ≡ -∇_θ G(π_θ)`；或
   - 离散策略改进：`π_* = argmin_π G(π)`，`无为 ⇔ max_π G(π) - min_π G(π) < τ`；
3. 为每个核心映射（道/一/德/观/明/同体大悲）建立“隐喻—模型”声明，明确其可测量代理、定义域和可证伪条件。

---

### 问题 2：L0-L7 频谱从“认知地形图”退化为“万能分类标签”

**问题本质**：L0-L7 是项目最有原创性的架构，但在应用中逐渐被当成事后分类工具。AI 被塞进 L4、创伤被塞进 L2、战争被塞进 L7，却**从未给出判定某一现象属于哪一层的可操作标准**。同时，频谱既说“层级编号不是价值高低”，又把 L0-L4 定义为健康、L5-L7 定义为负向轨迹，存在内在矛盾。

**影响**：任何人都可以把反对意见打发为“范畴错误”，频谱从可检验框架退化为解释一切的后见之明工具。

**建议**：
1. 为每一层制定**判定维度**：可共享性、可修正性、主体间验证方式、关系后果；
2. 允许同一现象跨多层评分（雷达图），而非强制单一层级归属；
3. 把“健康/负向”从层级定义中剥离，单独建立功能性评估标准（是否导致预测误差的有效更新、是否破坏关系）；
4. 设计“层级归属判准清单”，邀请独立评审者做编码一致性测试。

### 问题 3：关键模型量纲不一致、操作性定义缺失，形式化停留在纸面

**问题本质**：多个 ODE 和公式存在形式问题：

- `AB(t) = C_max - R_DMN(t)`：C_max 无独立测量，R_DMN 不是可相减的标量资源，且与后续文件中“DMN 也是全局觉知/放的神经基础”矛盾；
- `DMN-岛叶双稳态`切换条件：左侧是输入速率，右侧包含 sigmoid 激活阈值，量纲不一致；
- `100ms_model.md` 中第二组竞争方程稳态推导出现 `A^*` 同时出现在等式两边且量纲混乱；
- `SAB(t)` 和 `DMN-ECN 创造力模型`只给出“最大化条件”，未给出 `f` 的具体形式，无法闭合求解。

**影响**：这些模型无法进入数值仿真，也无法导出可检验的定量预测。项目所谓的“30+ 数学方程”大部分是“文字方程”。

**建议**：
1. 建立项目级 `NOTATION.md`，统一符号、定义域、量纲；
2. 对每个 ODE 做量纲分析，提供参数表、初始条件、典型取值范围；
3. 为每个核心模型创建可运行的 Python/Julia 仿真脚本，做参数扫描与分岔分析；
4. 对 `AB(t)` 给出标准化操作化定义，例如：
   ```
   AB(t) = 1 - [R_DMN(t) - R_0] / [R_max - R_0]
   ```
   其中 R_DMN 为 PCC/mPFC BOLD 相对基线的标准化变化。

### 问题 4：工程“最后一公里”断裂——站点与预印本均不可发布

**问题本质**：项目存在大量工程缺陷：

- `mkdocs.yml` 未启用 Mermaid 与数学公式扩展，部署后图表和公式无法正常渲染；
- 根目录 `README.md` 作为 MkDocs 首页会在严格模式下报错；
- 8 篇预印本中，preprint_4 至 preprint_8 为中文，却使用 `pdflatex` 编译，会失败；
- `requirements.txt` 未锁定版本，CI 不读取它；
- `.gitignore` 缺少 `site/`、LaTeX 辅助文件；
- 缺少 `LICENSE`、`CODE_OF_CONDUCT`、issue/PR 模板；
- GitHub Actions 只有部署、无 PR 质量检查。

**影响**：读者无法在 GitHub Pages 上正常阅读；学者无法编译预印本；贡献者缺乏规范；项目可信度受损。

**建议**：
1. 在 `mkdocs.yml` 启用 `pymdownx.superfences`（Mermaid）和 `pymdownx.arithmatex`（MathJax）；
2. 中文预印本改用 `ctexart` + `xelatex`；
3. 建立 `ci.yml`，对 PR 运行 `mkdocs build --strict`、链接检查、LaTeX 编译检查；
4. 补齐 LICENSE、CODE_OF_CONDUCT、issue/PR 模板；
5. 预编译 PDF 并提供下载链接。

## 三、独家视角：项目最深的机会与陷阱

### 3.1 你真正在建造什么？

Project Dao.Science 不只是“一本开源手册”。它有潜力成为一种**新型知识基础设施**：

> **第一人称认知科学的开源研究平台。**

这不是比喻。项目的核心命题——把东方心性实践与主动推理、神经科学对齐——天然要求一种混合方法论：
- 数学形式化（第三人称）
- 个人实修报告（第一人称）
- 神经影像/行为实验（第三人称验证）
- 工程实现（AI/教育/临床工具）

但目前，这四条腿中只有“数学形式化 + 概念叙述”比较发达，其余三条腿要么缺失，要么软弱。

### 3.2 最深的陷阱：“概念映射”替代“机制解释”

项目目前最大的潜在陷阱，是**用概念对应关系替代因果机制解释**。例如：
- “报冤行 ≈ 认知重评 + ACT 接纳”——但这只是标签对应，没有解释“为什么四行的组合比单独 CBT 更有效”；
- “知止 ≈ AI 停止标准”——但没有给出在 RL 代理中实现“知止”的具体元控制器架构；
- “五十阴魔 ≈ 缺氧导致前额叶失能”——过度简化，忽视了期望效应、文化脚本、睡眠剥夺等多因素。

**概念映射是起点，不是终点。** 项目需要从“这个古代概念像这个现代科学概念”进化到“这个古代实践通过什么神经/计算机制产生什么可测量效果”。

### 3.3 最深的深化方向：建立“可验证单元”

我建议项目引入一个核心产品：**每个核心命题对应一个“可验证单元”（Verifiable Unit）**，包含：
1. **形式化声明**（数学对象、定义域、边界条件）；
2. **仿真脚本**（可运行、可参数扫描）；
3. **实验方案草案**（被试、任务、指标、统计假设、预期效应量）；
4. **证据状态徽章**（F/S/B/N/M）；
5. **反事实条件**（什么证据会削弱或推翻该命题）。

这将把项目从“雄心勃勃的纲领”升级为“可运行的研究程序”。

### 3.4 独家战略判断

> **未来 6–12 个月，项目的决定性指标不是“写了多少新内容”，而是“能否让外部人独立验证或证伪项目的至少一个核心预测”。**

若能完成一个最小可行验证（例如：
- 一个 DMN-TPN 竞争模型的数值仿真；
- 一个 4 周“报冤行”练习的微型 RCT 方案与试点数据；
- 一个“知止”RL 元控制器在分布外任务上的行为验证），
项目将获得远超同等体量概念著作的学术可信度。

    - 在 CI 中自动生成 PDF 并上传 GitHub Release；

11. **跨领域试点网络**

    - 开源 Web 应用或 Notion 系统，追踪练习、量表、导出 CSV；

## 六、结语

Project Dao.Science 拥有一个罕见的组合：**深厚的东方心性传统、当代认知神经科学的前沿框架、以及开源共享的雄心**。它的最大价值不在于已经写下的 7300 行文字，而在于它有可能证明：**第一人称经验可以被严肃地、可检验地、集体地研究。**

但要抵达那里，项目必须完成一次痛苦的转型：从“写一本更好的书”转向“建造一个可被他人独立使用、验证和批评的基础设施”。

这不是内容问题，而是**认识论姿态问题**——你愿不愿意让每一个核心命题都暴露在被证伪的风险中？愿不愿意让项目从“作者的独白”变成“公共的实验”？

如果愿意，Project Dao.Science 将不只是“一个中文开源项目”，而会成为心智科学、AI 安全、临床干预和教育设计等多个领域都可以引用的基础设施。
