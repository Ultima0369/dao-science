# 贡献指南

## 分支策略
- `main`: 仅允许从 `develop` 合并，受保护
- `develop`: 集成分支，日常开发在此
- `feature/<名称>`: 新功能或新模块
- `doc/<名称>`: 纯文档修改
- `paper/<名称>`: 预印本相关

## 提交规范
遵循 Conventional Commits: `type(scope): description`
类型：feat, fix, docs, refactor, chore, paper

## 核心纪律
1. 所有内容需有科学与经典双重依据。
2. 第一人称实证案例必须匿名化。
3. 欢迎 pull request，但必须经过核心维护者审核。
