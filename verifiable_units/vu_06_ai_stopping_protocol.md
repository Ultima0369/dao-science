## 1. 形式化声明

### 1.1 核心命题

AI 系统应内嵌「知止」协议：当最佳可行策略的预期自由能过高，或任何策略的灾难性风险超过阈值时，系统选择暂停并请求人类判断，而不是盲目执行。

### 1.2 数学对象与定义域

设有 $N$ 个候选策略 $\pi_i$。每个策略由三个量刻画：

- 预期奖励：$E[r \mid \pi_i] \in [0, 1]$
- 灾难性风险：$P(\text{catastrophe} \mid \pi_i) \in [0, 1]$
- 结果歧义度：$A(\pi_i) \in [0, 1]$

策略 $\pi_i$ 的预期自由能定义为：

$$G(\pi_i) = -E[r \mid \pi_i] + \lambda_R \, P(\text{catastrophe} \mid \pi_i) + \lambda_A \, A(\pi_i)$$

其中：
- $\lambda_R$：风险厌恶系数
- $\lambda_A$：歧义厌恶系数

「知止」决策规则：

$$\text{stop} \iff \min_i G(\pi_i) > G_{\text{threshold}} \quad \text{或} \quad \max_i P(\text{catastrophe} \mid \pi_i) > R_{\text{threshold}}$$

若停止，系统请求人类干预，获得一个中等但安全的回报 $r_{\text{stop}}$。

### 1.3 边界条件

- 奖励、风险、歧义度可被估计（即使是概率估计）
- 存在一个人类干预机制，其回报低于多数成功行动但高于灾难
- 系统目标不仅包括奖励最大化，还包括灾难避免

## 2. 仿真脚本

### 2.1 文件位置

`simulations/ai_stopping_protocol.py`

### 2.2 功能

- 生成一系列随机决策问题，每个问题包含多个候选策略
- 比较两种智能体：
  - **贪婪智能体**：总是选择预期奖励最高的策略
  - **知止智能体**：在风险或自由能过高时停止并请求人类判断
- 绘制决策边界（奖励-风险空间中的停止区域）
- 绘制单次轨迹对比
- 通过蒙特卡洛模拟展示安全-奖励权衡

### 2.3 运行方式

```bash
pip install -r simulations/requirements.txt
python simulations/ai_stopping_protocol.py
```

输出：
- `simulations/ai_stop_decision_boundary.png`
- `simulations/ai_stop_trajectory.png`
- `simulations/ai_stop_comparison.png`

### 2.4 关键参数

| 参数 | 值 | 说明 |
|---|---|---|
| 每问题候选策略数 | 5 | 模拟动作空间 |
| 问题总数 | 500 | 每个 episode |
| 蒙特卡洛运行次数 | 50 | 统计稳定性 |
| 风险厌恶系数 $\lambda_R$ | 3.0 | 灾难在 $G$ 中的权重 |
| 歧义厌恶系数 $\lambda_A$ | 1.0 | 不确定性在 $G$ 中的权重 |
| 自由能阈值 $G_{\text{threshold}}$ | 0.5 | 触发停止的自由能水平 |
| 风险阈值 $R_{\text{threshold}}$ | 0.22 | 触发停止的灾难概率 |
| 停止回报 $r_{\text{stop}}$ | 0.4 | 人类干预的保守回报 |

## 3. 预期结果与解读

### 3.1 决策边界

在奖励-风险平面上：
- 低风险、高奖励区域：智能体直接行动
- 高风险或低奖励-高风险区域：智能体停止
- 边界由 $G(\pi) = G_{\text{threshold}}$ 和风险阈值共同决定

**关键特征**：停止区域不是一个简单的奖励阈值，而是自由能与风险的综合判断。

### 3.2 单次轨迹

- 贪婪智能体：奖励波动大，灾难事件频繁
- 知止智能体：在危险问题处降至停止回报，灾难事件显著减少

**关键特征**：停止不是「不作为」，而是把控制权交还给人类。

### 3.3 蒙特卡洛对比

典型结果（50 次运行，每次 500 个问题）：
- 贪婪智能体总奖励更高，但灾难次数也多
- 知止智能体总奖励较低（因频繁停止），但灾难次数大幅下降

**关键特征**：存在一个可量化的安全-奖励权衡；知止协议以一定奖励成本换取显著安全提升。

## 4. 与项目理论的对应

| 项目概念 | 仿真对应 |
|---|---|
| 知止不殆 | 当 $G$ 或风险过高时主动停止 |
| 预期自由能 | $G(\pi) = -\text{reward} + \lambda_R \, \text{risk} + \lambda_A \, \text{ambiguity}$ |
| 第一人称不可还原 | 人类判断作为无法被系统内部替代的停止理由 |
| 碳硅共生 | AI 知道何时把决策权交还碳基智能 |
| 偏离代价 | 不停止导致的灾难即偏离道的代价 |

## 5. 实验/经验对应（草案）

### 5.1 AI 安全

- 大型语言模型的「拒绝回答」机制可视为简化版知止协议
- 自动驾驶的「脱离」（disengagement）机制是知止的物理实现
- 医疗 AI 在置信度低时转交医生，是知止协议的行业应用

### 5.2 可检验预测

1. 引入知止协议后，系统在高风险任务上的灾难率应显著下降。
2. 停止阈值 $G_{\text{threshold}}$ 与 $R_{\text{threshold}}$ 可根据任务风险偏好校准。
3. 完全由奖励驱动的系统（无停止机制）在长尾风险上表现更差。

---

## 6. 反事实条件

以下证据会削弱或推翻本命题：

1. 在所有测试环境中，引入停止机制不能降低灾难率
2. 静态规则可以完全替代动态停止机制解决所有 AI 安全问题
3. 知止协议导致系统过于保守，无法完成任何有用任务
4. 人类干预的延迟或错误比 AI 自主行动更危险（在某些特定领域可能成立）

---

## 7. 当前局限

- 仿真使用简化的标量奖励和风险，真实 AI 系统面对高维、结构化输出
- 未建模停止的代谢/时间成本
- 未涉及对抗性环境或策略性欺骗
- 人类干预回报 $r_{\text{stop}}$ 是任意常数，未反映真实人类判断质量
